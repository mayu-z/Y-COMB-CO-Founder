Phase 3 — Claude API Integration 🤖
Time: 2 days
Goal: Turn retrieved chunks into sharp, attributed answers

Step 1 — Get your API key
From console.anthropic.com — store it as an environment variable, never in your code.
Step 2 — Write your system prompt
This is critical. Your system prompt should tell Claude:

You are a YC startup advisor
Answer only using the provided context
Always attribute — say where the insight comes from
If the context doesn't support the answer, say so explicitly — never invent
Be direct and specific, not generic
Reference real company examples when available in the context

Step 3 — Build the answer generator
Takes the user question + 5 retrieved chunks → sends to Claude → returns answer
Step 4 — Add a fallback handler
If retrieved chunks have low similarity scores — meaning the question is probably out of scope — don't send to Claude at all. Return a canned response: "I don't have reliable YC data to answer this well. Try asking about startup strategy, YC applications, or founder advice."
Step 5 — Test with 30 questions
Cover every question type — philosophical, application, pattern matching, company lookup, fundraising, hiring, PMF. Note where answers feel weak and trace it back to either chunking or retrieval issues.
✅ Phase 3 done when 80%+ of your test questions get sharp, attributed, accurate answers.
