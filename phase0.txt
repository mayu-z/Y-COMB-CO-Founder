YC Co-Founder RAG — Full Build Roadmap

Phase 0 — Data Collection 🌐
Time: 3–4 days
Goal: Build your raw data library before touching any code
This is the most important phase. Everything downstream depends on data quality.

Source 1 — Paul Graham Essays
URL: paulgraham.com/articles.html
How to retrieve:

Use requests + BeautifulSoup in Python
The articles page lists every essay as a link — loop through each one, fetch the page, extract the main text body
Save each essay as its own .txt file named after the essay title
There are about 220 essays — this will take a few minutes to scrape

What to watch:

Scrape slowly — add a 2 second delay between each request
Some essays have footnotes — keep them, they add context
Don't strip punctuation — PG's writing style carries meaning

Priority essays to make sure you have:

Do Things That Don't Scale
How to Get Startup Ideas
Default Alive or Default Dead
Hiring is Obsolete
How to Convince Investors
What Investors Look For


Source 2 — YC Blog
URL: ycombinator.com/blog
How to retrieve:

Use requests + BeautifulSoup
Blog index page lists all posts — loop through, extract title + full post body + author + date
Save as individual .txt files with metadata in the filename — 2023-dalton-how-to-apply.txt
Focus on posts by YC partners — Dalton Caldwell, Michael Seibel, Jared Friedman, Gustaf Alströmer

What to watch:

Some posts are short announcements — skip anything under 300 words, not useful for RAG
Keep author name in the saved file — you'll use it for attribution later
About 200–300 useful posts total


Source 3 — YC Startup School
URL: startupschool.org/curriculum
How to retrieve:

Startup School lectures are on YouTube — get the transcript for each video
Use youtube-transcript-api Python library — it pulls auto-generated or manual transcripts
Each lecture is about 20–40 minutes — transcripts will be long, that's good
Save each as its own .txt file with the lecture title and speaker name

Key lectures to prioritize:

How to Talk to Users — Gustaf Alströmer
How to Find Product Market Fit — David Rusenko
How to Build and Sell — Michael Seibel
Fundraising — Kirsty Nathoo
How to Apply to YC — Dalton Caldwell

What to watch:

Transcripts have filler words — "um", "uh", "you know" — clean these out with a simple string replace before saving
Speaker labels in transcripts are useful — keep them


Source 4 — YC Company Directory
URL: ycombinator.com/companies
How to retrieve:

YC's company page is JavaScript rendered — requests alone won't work
Use Playwright or Selenium to load the page fully, then extract
Each company card has — name, batch, industry, region, one-line description, website, status
Alternatively — the Kaggle YC dataset covers most of this, use that as your base and supplement with scraping for missing entries
Save as a single companies.csv

What to watch:

This page has thousands of companies — paginate carefully
Some companies are stealth and have no description — skip them
Status field matters — tag every company as Active, Acquired, or Inactive


Source 5 — Hacker News YC Threads
URL: Use HN Algolia API — hn.algolia.com/api
How to retrieve:

HN has a free public API — no scraping needed, just API calls
Search for threads tagged Ask HN with keywords like "YC application", "YC interview", "got into YC", "rejected by YC"
Also pull all Launch HN posts — these are founders describing their own YC-backed startups
Filter to only keep comments/posts with more than 5 upvotes — quality filter
Save as hn_threads.json

What to watch:

HN API is free and generous — but still add delays between calls
A lot of HN content is noise — be aggressive with your upvote filter
Launch HN posts are gold — founders explaining their own product in plain language


Source 6 — YC Application Questions
These are publicly known — no scraping needed
Just manually create a file called yc_application_questions.txt with all the known YC application questions. They're widely published and discussed. This file becomes important context when users ask "how do I apply to YC."
